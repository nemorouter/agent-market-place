Ask AI Guru — Docs

A pluggable AI support agent. Embed the hosted one in seconds, or run your own — your docs, your brand — all powered by the Nemo Router gateway (one key, one bill).

1 · Embed the widget 30 seconds

Drop one line before </body> on any page. It injects a bottom-right launcher that opens the chat — the sk-nemo key never reaches the browser (the widget talks to its own backend in an iframe).

<script src="https://guru-cs-agent-suz5ioxcsq-uc.a.run.app/widget.js"></script>

That's the hosted “Ask AI Guru” (answers about Nemo Router). To brand it as your own and answer from your docs, run your own agent below.

2 · Run your own agent

The widget is one open-source app you fork. You own the frontend, backend, and vector DB; Nemo Router is the brain (LLM + embeddings).

a. Fork & install

git clone <your-fork> my-agent && cd my-agent
npm install

b. Create a virtual key (the agent's identity + budget)

In the Nemo dashboard → Keys → Create virtual key. Set a per-day budget — that's your hard spend cap (covers chat and embeddings). Copy the sk-nemo-….

c. Point at your storage + configure

# .env.local
NEMOROUTER_API_KEY=sk-nemo-...
MODEL=gemini-2.5-flash-lite
EMBEDDING_MODEL=text-embedding-005        # Vertex, 768-dim
SUPABASE_URL=https://YOUR-PROJECT.supabase.co
SUPABASE_SERVICE_ROLE_KEY=...             # (or SUPABASE_ANON_KEY for read-only retrieval)
SUPABASE_SCHEMA=public
SYSTEM_PROMPT=You are MyCo's assistant. Answer only from the provided context.
ALLOWED_ORIGINS=https://myco.com,https://*.myco.com,http://localhost:3000
ADMIN_TOKEN=<long-random>

d. Seed the knowledge base + run

psql "$DATABASE_URL" -f supabase/migration.sql   # pgvector KB + RLS
npm run dev                                      # http://localhost:3000
npm run ingest                                   # crawl/ingest your docs → embeddings → KB

e. Deploy (one command per env)

npm run deploy:prod      # Google Cloud Run today; CLOUD=azure/aws tomorrow

Customize the launcher brand with data-title on the script tag, or set NEXT_PUBLIC_AGENT_NAME at build for the hosted page.

3 · Tracking costs

Every call — chat and embeddings — goes through the Nemo gateway on your virtual key, so all of it is metered, headered, logged, and budget-capped. See track-amp-nemo-costs for the full guide. The short version:

WhereWhat you get
Response headerx-nemo-request-cost on every chat + embedding response
LiteLLM_SpendLogsper-request ledger: call_type, model, spend
LiteLLM_VerificationToken.spendaggregate key spend — enforces the budget

Example: cost split by model (chat vs embeddings)

select model, count(*) as calls, round(sum(spend)::numeric, 6) as usd
from "LiteLLM_SpendLogs"
where "startTime" > now() - interval '24 hours'
group by model order by usd desc;
-- vertex_ai/gemini-2.5-flash-lite  | 8  | 0.001271   (chat)
-- vertex_ai/text-embedding-005     | 99 | 0.000637   (embeddings — captured!)

Embeddings are real line items (one per ingested chunk + one per question). They draw down the same key budget as chat — nothing is free or hidden.
